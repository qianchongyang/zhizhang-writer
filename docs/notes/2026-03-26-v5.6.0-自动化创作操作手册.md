# v5.6.0 自动化批量创作操作手册

**版本**: v5.6.0
**更新日期**: 2026-03-26
**适用项目**: webnovel-writer 插件

---

## 一、功能概览

v5.6.0 新增**自动化批量创作**能力，支持一键连续写作多章节，并提供夜间模式、质量门控、异常恢复等功能。

### 核心新增功能

| 功能 | 说明 |
|------|------|
| **BatchWriter** | 批量写作控制器，自动连续执行多章节写作 |
| **夜间模式** | `--night-mode` + `--max-calls`，限制 AI 调用次数，适合夜间无人值守 |
| **质量门控** | `--min-quality-score`，章节得分低于阈值时自动停止 |
| **断点恢复** | `--resume`，写作中断后可从断点继续 |
| **批量写入** | `batch-write` 接口，多章节状态批量回写 |

---

## 二、快速开始

### 2.1 单章写作（传统方式）

```bash
cd /path/to/你的小说项目
/webnovel-write 10
```

### 2.2 批量写作（自动化）

```bash
# 从第10章写到第40章
python3 -X utf8 "${CLAUDE_PLUGIN_ROOT}/scripts/webnovel.py" \
  --project-root "/path/to/你的小说项目" \
  batch run \
  --from 10 \
  --to 40
```

### 2.3 批量写作 + 夜间模式

限制 AI 调用次数，适合夜间无人值守：

```bash
python3 -X utf8 "${CLAUDE_PLUGIN_ROOT}/scripts/webnovel.py" \
  --project-root "/path/to/你的小说项目" \
  batch run \
  --from 10 \
  --to 40 \
  --night-mode \
  --max-calls 500
```

### 2.4 批量写作 + 质量门控

章节得分低于 80 分时自动停止：

```bash
python3 -X utf8 "${CLAUDE_PLUGIN_ROOT}/scripts/webnovel.py" \
  --project-root "/path/to/你的小说项目" \
  batch run \
  --from 10 \
  --to 40 \
  --min-quality-score 80
```

## 三、断点恢复

写作中断后，使用 `--resume` 从断点继续：

```bash
python3 -X utf8 "${CLAUDE_PLUGIN_ROOT}/scripts/webnovel.py" \
  --project-root "/path/to/你的小说项目" \
  batch resume
```

**注意**: `--resume` 会自动读取上次的进度文件，从中断的章节继续写作。

## 四、参数说明

### 4.1 batch run 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--from` | int | 必填 | 起始章节号 |
| `--to` | int | 必填 | 结束章节号 |
| `--night-mode` | flag | 关闭 | 启用夜间模式（限制 AI 调用次数） |
| `--max-calls` | int | 1400 | 夜间模式 AI 调用上限 |
| `--min-quality-score` | float | 75.0 | 最低质量分数阈值（低于此值自动停止） |

### 4.2 质量阈值参考

| 分数范围 | 质量等级 | 建议操作 |
|----------|----------|----------|
| 90-100 | 优秀 | 可直接发布 |
| 80-89 | 良好 | 可发布，有少量优化空间 |
| 70-79 | 中等 | 建议润色后发布 |
| 60-69 | 较差 | 需要大幅修改 |
| <60 | 很差 | 建议重写 |

## 五、工作流程

### 5.1 完整批量写作流程

```
1. 预检（确认项目正常）
   python3 scripts/webnovel.py --project-root "项目路径" preflight

2. 执行批量写作
   python3 scripts/webnovel.py --project-root "项目路径" batch run --from 10 --to 40

3. 检查结果
   - 查看 "审查报告/第NNN-NNN章审查报告.md"
   - 查看 ".webnovel/summaries/" 目录的章节摘要
```

### 5.2 中断后恢复流程

```
1. 检查中断状态
   python3 scripts/webnovel.py --project-root "项目路径" workflow status

2. 从断点继续
   python3 scripts/webnovel.py --project-root "项目路径" batch resume
```

## 六、输出产物

批量写作完成后，自动生成以下文件：

| 产物 | 位置 | 说明 |
|------|------|------|
| 章节正文 | `正文/第NNNN章-{标题}.md` | 润色后的可发布章节 |
| 审查报告 | `审查报告/第NNNN-NNNN章审查报告.md` | 综合审查结果 |
| 章节摘要 | `.webnovel/summaries/chNNNN.md` | 供后续章节消费的摘要 |
| 状态文件 | `.webnovel/state.json` | 更新的角色/势力/物品状态 |
| 索引文件 | `.webnovel/index.db` | 更新的向量索引 |

## 七、常见问题

### Q1: 批量写作中途停止了怎么办？

使用断点恢复：
```bash
python3 scripts/webnovel.py --project-root "项目路径" batch resume
```

### Q2: 夜间模式用完了怎么办？

默认 `--max-calls 1400`，按每章约 20-30 次调用估算，约可写 50-70 章。额度用完后会自动停止，次日继续写作时从断点恢复即可。

### Q3: 某章得分很低但想继续写怎么办？

取消质量门控，或调低阈值：
```bash
python3 scripts/webnovel.py --project-root "项目路径" batch run \
  --from 10 --to 40 --min-quality-score 60
```

### Q4: 如何查看当前进度？

```bash
python3 scripts/webnovel.py --project-root "项目路径" workflow status
```

## 八、最佳实践

1. **首次批量写作**：先跑 3-5 章测试，观察平均得分和 AI 消耗
2. **夜间模式**：建议 `--max-calls 500-800`，留有余量
3. **质量门控**：首次写作建议用默认值 75 分，确认流程稳定后可提高
4. **定期检查**：每隔 10 章检查一次输出质量
5. **及时备份**：定期提交 git，保留历史版本
