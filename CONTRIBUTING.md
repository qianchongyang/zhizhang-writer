# Contributing to 织章

感谢你愿意给织章提交 PR。

这份说明用于降低沟通成本，避免把个人偏好和项目规范混在一起。

## 提交前先确认

- 你的改动是否符合 `织章` 的独立品牌定位
- 你的改动是否与当前命令兼容策略一致
- 你的改动是否会影响新手 3 步安装路径
- 你的改动是否会引入密钥、私聊记录、个人资料或其他敏感信息

## 推荐流程

1. 先开 Issue 说明问题或方案
2. 再创建分支
3. 只做一个主题的改动
4. 补必要文档和测试
5. 提交 PR 并说明影响范围

## 分支建议

- `feature/<short-topic>`
- `fix/<short-topic>`
- `docs/<short-topic>`

## PR 要求

- 标题简短明确
- 说明改了什么、为什么改、怎么验证
- 如果是兼容迁移，说明旧命令是否仍保留
- 如果是破坏性改动，提前说明迁移成本

## 代码与文档原则

- 保留 GPL-3.0 兼容性
- 保留原始来源说明
- 新内容优先使用 `zhizhang` 命名
- 旧命名可作为兼容层，不建议直接删除
- 文档优先于实现，先让用户看懂，再让用户用顺

## 不接受的内容

- 密钥、token、证书、账号密码
- 个人聊天记录
- 私有项目内部讨论
- 与本项目无关的大型重构
- 无说明、无测试、无上下文的代码堆叠

## 我们怎么合并

维护者通常会优先合并：

- 小而清楚的修复
- 文档补充
- 安装与兼容性改进
- 能降低新手门槛的改动

如果你的 PR 很大，建议拆成多个小 PR。
