## 需求描述

### 问题背景

当前插件（webnovel-writer）从插件市场缓存目录加载规则：
```
~/.claude/plugins/cache/webnovel-writer-marketplace/webnovel-writer/5.5.4/
```

但开发者（用户）在源码目录修改规则：
```
/Users/zhoukai/code/webnovel-writer/webnovel-writer/
```

两者不同步，导致修改不生效。

### 用户期望

1. 修改源码目录的规则后，插件自动加载最新规则，无需手动同步
2. 支持多本小说项目，每本可独立指定规则源码
3. 不依赖插件市场更新流程

---

## 解决方案

### 方案：配置文件 + 自动加载

**步骤1**: 在小说项目根目录创建 `.webnovel/rules-source.txt`
```bash
# 内容为规则源码目录路径
/Users/zhoukai/code/webnovel-writer/webnovel-writer/skills
```

**步骤2**: 修改插件启动逻辑
- 插件启动时检查小说项目的 `.webnovel/rules-source.txt`
- 如果文件存在且有内容，从指定目录加载规则
- 否则回退到插件缓存目录

### 实现位置

- 插件启动脚本或 `webnovel.py`
- 检查 `PROJECT_ROOT/.webnovel/rules-source.txt`

### 验收标准

- [ ] 修改源码目录规则后，重新运行审查器能读到最新规则
- [ ] 多本小说可指定不同规则源码目录
- [ ] 不指定时默认使用插件缓存（向后兼容）
