# 动态大纲调整机制

**日期**: 2026-03-26
**目标**: 解决大纲生成时可能存在错误，以及写作过程中需要添加新情节的问题

---

## 问题场景

### 场景1: 大纲错误
```
大纲第13章: 筑基丹被偷
大纲第35章: 获得筑基丹（冠军奖励）

问题: 练气三层的吴彤在前12章不可能有筑基丹
```

### 场景2: 需要增强某个情节
```
当前大纲: 宗门大比只有2-3章
用户期望: 宗门大比需要10章的小高潮副本
```

### 场景3: 章节内容不完整
```
写完第15章后，发现某个情节点没有写清楚
需要补充/重写某个情节
```

---

## 解决方案

### 方案A: 大纲冲突检测（自动检测）

在 `/webnovel-write` 执行时，自动检测：

1. **状态一致性检查**
   - Data Agent 提取的状态（character_states, item_states）与历史记录比对
   - 如果检测到矛盾，生成警告

2. **时间线逻辑检查**
   - 当前章节的时间 vs 历史时间线
   - 检测时间逆流、物品凭空出现等异常

3. **冲突报告格式**
   ```
   ⚠️ 大纲冲突检测:
   第13章大纲说"筑基丹被偷"
   但历史记录显示: 筑基丹将在第35章才获得
   建议: 修改第13章大纲为"灵石被盗"
   ```

### 方案B: 动态大纲调整指令

新增命令/指令：

```bash
/webnovel-adjust --chapter 13 --reason "筑基丹来源错误" --new-plot "灵石被盗"
```

或通过对话：

```
用户: 第13章的大纲有问题，筑基丹应该是第35章才获得
Claude: 明白，我将第13章的大纲修改为"灵石被盗（实验材料）"
```

### 方案C: 副本/支线插入机制

```bash
/webnovel-insert --arc "宗门大比副本" --start-chapter 20 --length 10 --type "副本"
```

副本特征：
- 有独立的起承转合
- 占用指定章节数
- 自动插入到主时间线中
- 后续章节编号自动调整

### 方案D: 章节增强指令

```bash
/webnovel-enhance --chapter 15 --reason "打斗场面不够精彩" --method "添加副本"
```

## 实现思路

### 触发时机

| 时机 | 检测内容 |
|------|---------|
| `/webnovel-write` 执行前 | 大纲冲突检测 |
| Data Agent 处理时 | 状态一致性检查 |
| Consistency Checker 审查时 | 时间线/逻辑冲突 |
| 用户主动触发 | 手动调整 |

### 调整类型

1. **小修**: 修改某章的某一情节（如"筑基丹被偷"→"灵石被盗"）
2. **中修**: 插入新章节（如在第20-30章之间插入宗门大比副本）
3. **大修**: 重写整个卷的大纲（如调整第一卷的核心冲突）

### 文档变更

调整后自动更新：
- `大纲/第X卷大纲.md`
- `大纲/时间线.md`
- `state.json` 中的时间线记录
