# webnovel-writer v5.7.0 升级设计

**日期**: 2026-03-27
**版本**: v5.7.0
**类型**: 功能增强

---

## 概述

借鉴 chinese-novelist-skill 的优秀设计，对 webnovel-writer 系统进行三个方向的升级：

1. 字数检查系统
2. 质量报告总分层
3. references 参考资料扩展

---

## Phase 1: 字数检查系统

### 目标

在写作流程中加入字数验证，确保章节交付长度符合标准。

### 设计

| 检查点 | 规则 |
|--------|------|
| 字数标准 | 每章 2000-2500 字（可配置） |
| 写作前提示 | Context Agent 输出"本章建议字数：XXXX字" |
| 审查时警告 | 低于 1800 字 → 警告 |
| 审查时强制 | 低于 1500 字 → MUST_FIX |
| 超出建议 | 超过 3000 字 → 建议拆分（非强制） |

### 实现文件

- `references/word-count-rules.md` (新增)
  - 字数标准定义
  - 各题材字数偏好
  - 字数异常处理指引

### 集成点

- `/webnovel-write`: Context Agent 输出字数建议
- `/webnovel-review`: 字数检查维度

## Phase 2: 质量报告总分层

### 目标

在六维并行审查独立输出的基础上，增加总分汇总层，提供统一验收标准。

### 设计

| 项目 | 内容 |
|------|------|
| 总分制 | 80 分制（可配置阈值） |
| 六维分项 | 独立输出（Hook/爽点/节奏/连贯/OOC/追读力） |
| 总分计算 | 各维度加权求和 |
| 交付标准 | ≥70 优秀/可直接交付，60-69 合格/轻微修改，<60 需修复或不合格 |

### 各维度权重

| 维度 | 权重 | 满分 |
|------|------|------|
| 追读力 (Reader Pull) | 20% | 16 分 |
| 爽点密度 (High-point) | 20% | 16 分 |
| 节奏 (Pacing) | 15% | 12 分 |
| 连贯性 (Consistency) | 20% | 16 分 |
| OOC 检查 | 15% | 12 分 |
| 叙事连贯 (Continuity) | 10% | 8 分 |
| **合计** | 100% | 80 分 |

### 交付等级

| 分数 | 等级 | 动作 |
|------|------|------|
| ≥70 | 优秀 | 可直接交付 |
| 60-69 | 合格 | 轻微修改后交付 |
| 50-59 | 需修改 | 主要问题需修复 |
| <50 | 不合格 | 必须大改 |

### 实现文件

- `references/checker-output-schema.md` (扩展)
  - 新增 `total_score` 字段
  - 新增 `grade` 字段
  - 新增各维度权重配置

## Phase 3: references 参考资料扩展

### 目标

将 chinese-novelist-skill 的优秀参考资料设计引入，扩展 references 系统。

### 新增文件

#### 3.1 `references/chapter-guide.md`

章节写作核心要点：

| 内容 | 说明 |
|------|------|
| 前 20% 法则 | 前 20% 决定生死，必须立即建立紧张感 |
| 10 种强力开头 | 行动中开场/反常情境/震撼对话/倒计时开场/危机时刻/背叛开场/重大选择/谜团浮现/重大发现/结局预告 |
| 标准结构 | 开头钩子(20%) → 发展推进(50-60%) → 高潮时刻(15-20%) → 结尾钩子(5-10%) |
| 节奏控制 | 紧张与缓解交替 |

#### 3.2 `references/hook-techniques.md`

悬念设置技巧：

| 内容 | 说明 |
|------|------|
| 10 种经典悬念钩子 | 突然揭示/紧急危机/未完成动作/身份反转/两难选择/神秘物品/时间限制/承诺威胁/离奇消失/言外之意 |
| 悬念强度等级 | L1好奇 → L2关切 → L3迫切 → L4生存 → L5终极 |
| 章节间悬念连接 | 伏笔技巧/悬念升级/多线悬念 |
| 悬念设置禁忌 | 虚假悬念/机械降神/过度留白/低风险钩子 |

#### 3.3 `references/quality-checklist.md`

质量检查清单（80 分制）：

| 检查维度 | 分值 |
|----------|------|
| 基础要素（标题/字数/完整性） | 10 分 |
| 开篇钩子 | 10 分 |
| 情节推进（冲突/张力） | 15 分 |
| 人物一致性 | 10 分 |
| 对话有效性 | 10 分 |
| 悬念技巧 | 10 分 |
| 展示而非讲述 | 5 分 |
| 节奏控制 | 5 分 |
| 语言质量（去AI味） | 5 分 |
| 连贯性 | 10 分 |
| **合计** | 80 分 |

#### 3.4 `references/plot-structures.md`

情节结构模板：

| 模板 | 适用场景 |
|------|---------|
| 三幕结构 | 标准剧情 |
| 起承转合 | 中国古典叙事 |
| 英雄之旅 | 成长型故事 |
| 五段式 | 单元剧 |

#### 3.5 `references/dialogue-writing.md`

对话写作规范：

| 原则 | 说明 |
|------|------|
| 意图驱动 | 对话必须有目的（试探/回避/施压/诱导） |
| 性格体现 | 对话风格反映人物性格 |
| 信息传递 | 对话推进情节或揭示人物 |
| 避免说明 | 不用对话做纯说明 |

### core-constraints.md 扩展

引入四大核心法则：

```markdown
## 核心法则

1. **展示而非讲述** - 用动作和对话表现，不直接陈述
2. **冲突驱动剧情** - 每章必须有冲突或转折
3. **悬念承上启下** - 每章结尾必须留下钩子
4. **开头即高潮** - 前20%必须极其吸引人
```
