# Security Policy

织章是公开开源项目，但不接受任何形式的敏感信息提交。

## 不要提交

- `.env`
- API key / token / secret
- `credentials.json`
- `*.key`
- `*.pem`
- 个人聊天记录
- 私有复盘
- 内部调试截图中带有密钥的内容

## 发现安全问题怎么办

如果你发现的是：

- 密钥泄露
- 权限配置错误
- 敏感数据被写入仓库
- 可导致他人数据暴露的问题

请不要直接在公开 PR 里贴完整细节。

建议先：

1. 开 Issue 简要说明问题类型
2. 提供最小复现步骤
3. 避免贴出任何真实密钥或用户数据

## 维护者处理原则

- 先确认影响范围
- 再决定是文档修复、配置修复还是代码修复
- 涉及公开泄露的内容会优先处理

## 用户自查建议

提交前请确认：

- `.gitignore` 已覆盖本地密钥和私有目录
- `README.md` 示例中的地址和账号是占位符
- `docs/` 里没有聊天记录、个人笔记或客户信息

如果你不确定某个文件能不能公开，默认按“不能公开”处理。
