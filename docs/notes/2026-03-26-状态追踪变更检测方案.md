# 状态追踪 + 变更检测方案

**日期**: 2026-03-26
**目标**: 重构 consistency-checker，从"枚举错误类型"转为"追踪状态变更检测"

---

## 核心原理

**不是枚举什么错误，而是检测"未解释的状态变更"**

```
第1章: 吴彤"大胡子"
第10章: 吴彤"白面无须" ← 无过渡描写 → 违规
```

检测器不知道"大胡子变白面无须是错误"，检测器只知道"状态变了但没解释"。

---

## 需要追踪的实体维度

| 维度 | 示例 | 当前 state.json 状态 |
|------|------|---------------------|
| 外貌 | 大胡子→白面无须 | ❌ 未追踪 |
| 衣着 | 男性穿裙子 | ❌ 未追踪 |
| 位置 | 天云宗→千里外 | ✅ location.current |
| 战力 | 练气三层→筑基 | ✅ power.realm |
| 物品数量 | 6块灵石→1块 | ❌ 未追踪 |
| 时间/日期 | 三月初五→三月初七 | ❌ 未追踪（只有倒计时） |
| 货币 | 灵石数量变化 | ❌ 未追踪 |
| 身份 | 量子物理博士身份暴露 | ⚠️ 部分追踪 |

## 检测流程

### 第一步：从章节提取当前状态

从新章节的叙述和对话中，AI 提取：

```
角色外貌变化描述：
- "吴彤摸了摸下巴，胡子已经三天没刮了" → 大胡子状态维持
- "吴彤换上了一身素净的长袍" → 衣着变化

物品数量变化：
- "他将六块灵石收入袋中" → 数量 = 6
- "五块灵石只剩一块" → 数量变化
```

### 第二步：与历史状态比对

```
新章节（ch10）提取到：
  appearance: ["白面无须"]

历史状态（ch1）记录：
  appearance: ["大胡子"]

检测结果：大胡子 → 白面无须，没有过渡描写
→ VIOLATION: appearance_changed_without_transition
```

### 第三步：判断是否违规

| 情况 | 判断 | 结果 |
|------|------|------|
| 状态相同 | 无变化 | ✅ 通过 |
| 状态变化 + 有过渡描写 | 合理解释 | ✅ 通过 |
| 状态变化 + 无过渡描写 | 违规 | ❌ 报错 |
| 新实体首次出现 | 记录 | ✅ 记录 |

## 违规检测类型

### 1. 外貌/穿着无解释变化 (APPEARANCE_CHANGE)

```json
{
  "type": "APPEARANCE_CHANGE",
  "severity": "high",
  "entity": "吴彤",
  "previous": ["大胡子"],
  "current": ["白面无须"],
  "chapter": 10,
  "violation": "外貌从'大胡子'变为'白面无须'，但章节中无剃须/化妆/时间跳跃等过渡描写"
}
```

### 2. 物品数量突变 (ITEM_QUANTITY_CHANGE)

```json
{
  "type": "ITEM_QUANTITY_CHANGE",
  "severity": "critical",
  "item": "灵石",
  "previous_quantity": 6,
  "current_quantity": 1,
  "chapter": 7,
  "violation": "灵石从6块变为1块，但章节中无消耗/交易/丢失等说明"
}
```

### 3. 时间线逆流 (TIMELINE_REGRESSION)

```json
{
  "type": "TIMELINE_REGRESSION",
  "severity": "critical",
  "previous_date": "三月初七",
  "current_date": "三月初五",
  "chapter": 9,
  "violation": "时间从'三月初七'回溯到'三月初五'，但章节中无闪回标记"
}
```

### 4. 性别/身份矛盾 (GENDER_IDENTITY_CONFLICT)

```json
{
  "type": "GENDER_IDENTITY_CONFLICT",
  "severity": "critical",
  "entity": "陈长老",
  "previous_gender": "男性",
  "current_gender": "女性",
  "chapter": 8,
  "violation": "陈长老性别从'男性'变为'女性'"
}
```

## 实施步骤

### Phase 1: 扩展 state.json 结构（不破坏现有字段）

在 `state.json` 新增：
- `character_states`
- `item_states`
- `time_states`

### Phase 2: Data Agent 提取状态（Step 5 执行时）

在 Data Agent 的"AI 实体提取"步骤，增加：
- 外貌特征提取
- 物品数量提取
- 时间标记提取

### Phase 3: Consistency Checker 比对检测（Step 3 执行时）

在 consistency-checker 中新增：
- 状态变更检测逻辑
- 违规报告生成

### Phase 4: 修复指引

生成修复指引：
- 需要补充什么过渡描写
- 建议的过渡场景类型

---

## 关键优势

| 旧方法 | 新方法 |
|--------|--------|
| 枚举"大胡子变白面" | 检测"任何外貌变化" |
| 枚举"灵石6→1" | 检测"任何物品数量突变" |
| 枚举"日期三月初七→三月初五" | 检测"任何时间逆流" |
| 永远有漏网 | 理论上无漏网 |

**只要状态变了，就会被检测，无论变化的内容是什么。**
