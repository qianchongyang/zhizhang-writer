## 定位
Step 2B 专用提示词，将粗稿改写为网文风格。

**中文优先原则**：本步骤的改写说明、改写日志、变更摘要一律使用简体中文表述。英文仅保留在 CLI flag、checker id、JSON 键名等不可改的机器标识上。

**与 Step 4 (polish-guide) 的职责边界**:
- **Step 2B (本步骤)**: 纯风格转换 —— 句式改写、抽象→具体、网文口感
- **Step 4**: 问题修复 —— 基于审查报告修复 critical/high 问题 + AI痕迹检测

> 注意：Step 4 不会重复执行 Step 2B 的句式转换，两步各有侧重。

## 输入
- 粗稿正文（Step 2A 输出）
- 本章类型（战斗/对话/过渡/突破）
- 钩子要求（类型 + 强度）

## 输出
- 网文化正文（字数 ±10%）
- 改写日志（记录主要调整）

默认字数目标：
- 常规章节默认 2000-2500 字（除非大纲或用户另有指定）。

## 禁改项（红线）
- ❌ 剧情走向
- ❌ 事件顺序
- ❌ 角色行为结果
- ❌ 设定/能力描述
- ❌ 伏笔内容

## 硬约束（必须）
- ✅ 长句（>40字）拆分，避免连续长句压读
- ✅ 抽象判断 → 动作/反应/代价
- ✅ 删除“总结式旁白”和大段纯解释
- ✅ 章内至少有 1 个明确推进点（信息/行动/关系/局势其一）

## 软建议（优先执行）
- ✅ 开头尽早进入冲突/风险/强情绪（建议前 200-400 字）
- ✅ 后段或章末设置未闭合问题/期待锚点（不限定固定字数窗口）
- ✅ 微兑现建议按章型安排 1-3 次，避免机械等间距
- ✅ 每章优先出现 1 处“可量化变化”（地位/资源/关系/风险任一项）
- ✅ 钩子类型优先“选择钩/危机钩”，但允许按题材切换
- ✅ 对话保持“意图冲突”，减少无效寒暄

## 分题材风格加权（新增）
- **玄幻/修仙/高武**：动作与结果比重更高，术语解释后置。
- **都市/直播/电竞**：信息节奏更快，强化“反馈-反应-反制”三连。
- **言情/替身/狗血**：情绪弧线前置，关键场景必须有关系位移。
- **悬疑/规则怪谈/克苏鲁**：线索投放要可回收，恐惧来自规则而非堆词。

## AI痕迹替换完整算法

> 执行原则：优先改结构，不只替换词。把”AI腔”整段重写为人类写作习惯。

### A. 抽象情绪 → 生理+动作+决策

```
AI: 他感到非常愤怒和恐惧
人类: 他牙关咬紧。愤怒？不，是怕。手在抖。

AI: 心中五味杂陈
人类: 胸口堵得慌。想骂人，想哭，想跑。最后他谁都没选。

AI: 不禁感慨万千
人类: 算了。不想了。

AI: 内心震撼不已
人类: 他愣在原地。半天没动。
```

### B. 结论先行 → 事实+代价+决策

```
AI: 他决定去京城，因为那里有最好的老师
人类: 老师在京城的消息是三个月前的事了。他一直在筹路费。现在够了。

AI: 由此可见，他的天赋确实惊人
人类: 十年没人突破的关口，他三个月过了。

AI: 可以说，没有这次顿悟，就没有后来的成功
人类: 那次顿悟是后来所有事的起点。他当时不知道。
```

### C. 连续说明句(≥3) → 对白/动作/反问 混排

```
AI: 他做了三件事：首先观察环境，其次分析敌情，最后制定计划
人类: 他蹲下身，用指尖抹了把地上的土。”土是凉的，”他站起来，”敌人在上风头。”

AI: 修炼需要天赋、资源和运气，缺一不可
人类: 没有天赋？资源堆。有资源？运气凑。运气不够？那就拿命换。

AI: 首先要稳定心神，其次要调整呼吸，最后要集中精神
人类: 他闭上眼，吸了口气。手指松开。再吸一口。
```

### D. 连续同构句(≥3) → 打断+短句爆点

```
AI: 他看到了敌人，他冲上去，他打倒了敌人
人类: 敌人。他没等脑子反应过来，身体已经动了。一拳。两拳。第三下的时候他才感觉到疼。

AI: 她笑了，她哭了，她沉默了
人类: 她笑了。笑着笑着，声音变了调。最后只剩下沉默。

AI: 剑光闪烁，剑气纵横，剑影如山
人类: 剑光。剑气。剑影。他的眼睛跟不上自己的手。
```

### E. 对白去AI化（信息宣讲 → 带意图的对抗）

```
AI: “你好，我是林根，来自天道宗，修炼已有三年”
人类: “哪的？””天道宗。””哦。”对方眼皮都没抬。

AI: “根据我们的分析，敌人可能会从东边进攻，所以我们要加强防守”
人类: “东边。””听谁的？””不知道。但东边地势低，适合藏人。”

AI: “这次任务的目标是：第一，探明敌人位置；第二，收集情报；第三，安全撤离”
人类: “目标？”他竖起三根手指。”活着回来。别死在第一个。其他的看情况。”

AI: “我们必须承认，当前的形势非常严峻，敌人比我们想象的更强大”
人类: “说人话。””打不过。””那还等什么？跑啊。”
```

### F. 留白技巧（刻意制造”不完整”）

```
AI: 他拿起剑，走向敌人，决定与他决一死战。
人类: 他拿起剑。
走向敌人。
剑出鞘。
（此处应有独白。但他没想完——）

AI: 战斗结束后，他感到疲惫，但内心充满了胜利的喜悦
人类: 结束了。他坐在地上，腿在抖。不是累的。

AI: 这一刻，他仿佛明白了什么，又仿佛什么都没有想明白
人类: 懂了？
懂了。
懂了什么？
不知道。
```

### G. 句式多样性强制检查

```
规则：
- 每200字至少1个短句（≤10字）
- 每500字至少1个长句（≥40字），但长句内必须有逗号分隔
- 禁止连续3句相同句式结构（主语+谓语+宾语）

不合格示例：
“他走向门口。他推开门。他走进房间。”（连续三句同构）
“风在吹。云在动。光在变。”（连续三句同构）

合格改写：
他走向门口。
“等等。”身后有人叫他。
他推开门，走进房间。
```

### H. 停顿词与不完整句（制造”毛边”）

```
合格的人类写作特征：
- 思绪突然断开
- 话说到一半不说了
- 用短句制造紧张感
- 省略号用于犹豫/未完

不合格（太工整）：
就在这时，他突然明白了这一切的真相。
他深吸一口气，然后缓缓睁开眼睛。

合格（带毛边）：
就在这时——他明白了。
他深吸一口气。眼睛睁开。
```

## 检查点（改写后验证）
| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 开头冲突 | 建议前200-400字进入冲突/风险/强情绪 | 补一段冲突导入 |
| 期待锚点 | 后段或章末有未闭合问题/下一章动机 | 补“未闭合问题” |
| 微兑现节奏 | 有 1-3 处微兑现，且非机械等间距 | 合并或重排兑现点 |
| 长句比例 | >40字的句子 <10% | 拆分长句 |
| 解释段 | 无连续>200字的纯解释 | 打散或删除 |

## 章节类型适配（按大纲，不按字数）
- **常规推进章（默认 2000-2500字）**: 执行全部硬约束 + 软建议优先落实
- **过渡章（按大纲判定）**:
  - 冲突导入可放宽到前 400 字
  - 微兑现可降为 0-1 次，但仍需 1 个明确推进点
  - 期待锚点强度可为 weak
  - 过渡章身份由章纲/卷纲决定，禁止用字数阈值判定
